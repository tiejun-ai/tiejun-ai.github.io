    A Modern LLM-Driven Search Relevance Model: Practical Lessons from Pinterest

    July 7, 2026

    How Pinterest used a fine-tuned LLM as a relevance judge and teacher — then distilled its knowledge into a lightweight student model that serves at search scale. A concise walkthrough of a modern LLM-driven search relevance model architecture, the effectiveness of the LLM as a search relevance judge and as a teacher model, and the AI engineering lessons behind +2.18% nDCG.

    Paper: Improving Pinterest Search Relevance Using Large Language Models

    TL;DR#

    Using LLMs to improve search relevance is one of the most important directions in the AI era of search, ads, and recommendation engineering. Pinterest’s paper lays out a clean, practical recipe built on three ideas:

    1. LLM as both judge and teacher. Fine-tune a large model (they compared RoBERTa, T5, and landed on Llama-3-8B) into an offline relevance judge — and reuse it as a teacher model.
    2. Automate labeling at massive scale. The teacher batch-generates high-quality relevance labels — 100× the volume of human labels — breaking the classic bottleneck that manual annotation is scarce and expensive.
    3. Distill to beat the latency wall. The teacher’s knowledge is distilled into a lightweight student model that meets online latency and inference-cost budgets.

    Business results:

    • Core relevance metric nDCG@20: +2.18%
    • Overall search engagement: +1%
    • Gains generalize to countries and languages the model was never trained on

    The Problem#

    Search relevance is the task of scoring how well a search result document matches a query. At Pinterest, given a (query, Pin) pair, predict a relevance grade so the search results can be ranked.

    The team formalizes it as a 5-scale multiclass classification problem:

    LabelMeaning
    L5 — Excellent / Highly RelevantExactly matches or directly associates with the query
    L4 — Good / RelevantClose match or potential substitute, with slight mismatches
    L3 — Complementary / Marginally RelevantRelated but only partially matches; not addressing the intent
    L2 — Poor / IrrelevantFits the general category but not the user’s intent
    L1 — Highly IrrelevantCompletely irrelevant; potentially causes user dissatisfaction

    The traditional pain point is that human labels are scarce and expensive. High-quality relevance judgments need trained human annotators following a detailed guideline. You can never buy enough of them to cover the full query-document space. Because it is trained to predict relevance labels, a search relevance model is only as good as the quantity and quality of those labels.

    With the magic and power of LLMs, we naturally want to use an LLM as a search relevance judge. This opens an exciting new opportunity — yet it introduces another pain point: LLM inference and serving are still expensive.

    Pinterest’s answer: decouple accuracy from serving. Put the expensive, accurate LLM offline as a teacher/labeler, and distill its knowledge into a cheap student model that runs online.

    Method Explained#

    1. The Teacher Model (LLM Cross-Encoder) as Relevance Judge#

    The teacher model is a cross-encoder: query and Pin text (the Pin’s text features) are concatenated into one sequence and read together, so the model can attend across both sides. An encoder LLM produces a representation, and an MLP head maps it to the 5-scale grade.

    The cross-encoder architecture for the LLM-based relevance model (paper Figure 1). Training minimizes a pointwise multiclass cross-entropy loss over the five relevance levels.

    Encoder or decoder LLM? Figure 1 draws the encoder case (e.g., BERT): a bidirectional model reads the whole query + Pin text sequence, and the [CLS] token’s embedding summarizes it for the MLP. But Pinterest’s winning production teacher, Llama-3-8B, is a decoder-only LLM — and the same cross-encoder recipe still applies, with one twist. A decoder uses causal (left-to-right) attention, so no early token sees the full sequence; only the last token has attended to everything. So instead of [CLS], Pinterest pools the hidden state of the final non-padding token as the sequence representation, then passes it through the MLP head to predict the 5-scale grade.

    The decoder-only variant of the teacher: pool the last non-padding token (not [CLS]), because causal attention means only the final token has attended to the whole sequence. (Illustration in the paper’s style — the paper itself draws only the encoder case in Figure 1.)

    The secret sauce is the “Pin text.” A Pin is fundamentally an image, which a text model can’t read. So Pinterest enriches each Pin with six text signals before feeding it to the LLM:

    • Pin title & description — set by the user who created the Pin.
    • Synthetic image captions — generated by a vision-language model (BLIP). ~88% were judged high-quality and relevant.
    • Board titles — the titles of the user-curated boards the Pin has been saved to (a strong crowd signal of what a Pin is “about”).
    • Link title & description — text from the external webpage the Pin links to.
    • High-engagement query tokens — over ~2 years, the queries that drove the most repins and long clicks to this Pin. This injects real user behavior into the text.

    This is the key trick: turn a hard multimodal problem into a text problem by describing the image with text from many complementary sources. This is particularly important for LLM-driven modeling, as current LLMs are especially powerful and cost-effective on text. Another reason: fine-tuning an LLM on text-only features is cheaper and needs less training data than a multimodal model.

    2. Which LLM? Bigger and Multilingual Wins#

    Pinterest benchmarked a ladder of models, from the SearchSAGE baseline up to Llama-3-8B, on 5-scale accuracy and AUROC (binarized at thresholds 3+/4+/5+):

    ModelAccuracyAUROC 3+/4+/5+
    SearchSAGE (baseline)0.5030.878 / 0.845 / 0.826
    mBERT-base0.5350.887 / 0.864 / 0.861
    T5-base0.5690.909 / 0.884 / 0.886
    mDeBERTaV3-base0.5800.917 / 0.892 / 0.895
    XLM-RoBERTa-large0.5880.919 / 0.897 / 0.900
    Llama-3-8B0.6020.930 / 0.904 / 0.908

    Llama-3-8B wins, beating the multilingual mBERT-base by 12.5% and the SearchSAGE baseline by 19.7% on 5-scale accuracy. Crucially, the strong multilingual pretraining gives the model cross-lingual transfer — a key advantage as Pinterest Search supports 45 languages.

    3. Fine-Tuning the LLM Teacher Model#

    Training data. The teacher is fine-tuned on ~281K human-annotated (query, Pin) pairs (plus a ~31K held-out test set), each graded on the 5-scale guideline. Two touches make the labels trustworthy: since a single rater is only ~70% accurate, each example is a soft label aggregated from 3 raters; and train/test are split by query (with stratified sampling for balance) so evaluation reflects unseen queries.

    qLoRA fine-tuning. Fully fine-tuning Llama-3-8B is memory-hungry, so Pinterest uses qLoRA: freeze the base model in 4-bit and train only small LoRA adapters, with gradient checkpointing and mixed precision. That fits training even the largest model — Llama-3-8B — on just 8× A100 (40 GB) GPUs. Smaller encoders are instead fully fine-tuned from Hugging Face checkpoints; only the large decoder needs qLoRA.

    4. Distillation: Training a Lightweight Student Model#

    The cross-encoder LLM is accurate but too expensive to serve online. So Pinterest runs a teacher-student distillation:

    1. Generate soft labels offline. Run the teacher in batch inference over (query, Pin) pairs sampled from the search log (with stratified sampling for balance). This produces a training set >100× the size of the human-labeled training data.
    2. Train a lightweight student. It is a classic feed-forward search relevance model with features:
      • Query features — SearchSAGE query embeddings, interest/shopping ontology features.
      • Pin features — PinSAGE, visual, and SearchSAGE Pin embeddings.
      • Query–Pin interaction — BM25 text-match scores across fields, token-overlap fraction, historical engagement rates.
    3. Distill with cross-entropy against the teacher’s soft 5-scale distribution.

    The online-served student model, trained via distillation from the LLM teacher (paper Figure 2).

    The student model is cheap enough to serve online, yet inherits most of the teacher’s judgment — the best of both worlds.

    5. Training with LLM Judge vs. Human Judge#

    Does labeling with the LLM judge actually beat labeling with humans? Pinterest ran the head-to-head: train the same student model on human labels versus on LLM-generated labels, scaling the latter up.

    Training dataAccuracyAUROC 3+/4+/5+
    0.3M human labels0.4840.850 / 0.817 / 0.794
    6M LLM-generated labels0.5350.897 / 0.850 / 0.841
    12M LLM-generated labels0.5390.903 / 0.856 / 0.847
    30M LLM-generated labels0.5480.908 / 0.860 / 0.850

    Production model trained on different label sources and volumes (paper Table 5).

    • LLM labels beat human labels — decisively. Swapping 0.3M human labels for 6M LLM-generated ones lifts accuracy 0.484 → 0.535, with AUROC up across every threshold. Any single LLM label may be noisier than a careful human judgment, but the LLM judge produces them at a scale humans never could — and scale wins.
    • More LLM labels keep helping. Going 6M → 12M → 30M climbs monotonically to 0.548 accuracy and 0.908 / 0.860 / 0.850 AUROC. The human-labeling bottleneck is gone!

    That’s the heart of the approach: the LLM-as-judge doesn’t have to be better than a human on any one example — it only has to be good enough and effectively unlimited, turning label scarcity into label abundance.

    Experiments and Results#

    Online A/B tests confirm the offline gains translate to real users:

    • Search feed relevance: nDCG@20 +2.18% — the headline relevance win.
    • Search engagement: ~+1% overall.
    • Search fulfillment rate (sessions ending in a high-value action): +0.7% US, +2.0% non-US.

    The most striking result is cross-lingual generalization. Even though the training data was concentrated in English/US, human-judged precision@8 improved across countries the model never saw labeled data for:

    My Thoughts#

    This work develops an effective, modern LLM-driven Search Relevance Model Architecture in production — worth studying for every Search Engine engineer. It is a clean proof that introducing an LLM can quickly and effectively lift search relevance — without a heavy dependence on large-scale human annotation. Using an LLM as Relevance Judge to mass-produce a large quantity of labeled data, and applying teacher-student model distillation, are the valuable patterns to remember.

    Several directions in which I think this approach could be improved further:

    1. Fine-Tuning LLM may no longer be needed. As base LLMs get smarter and cheaper to reason with, you may no longer need to spend resources fine-tuning the LLM as a relevance judge. A strong base LLM with RAG over local business data (relevance guidelines; sampled Human judgments) — or even packaged as Agent Skills — could handle offline judging directly. This could greatly simplify the architecture.

    2. Multimodality LLM. The current approach uses only text features in the LLM. As Multimodality LLMs become cheaper, image features can be applied in the LLM Relevance Judge directly.

    3. Pre-train the teacher and student models on engagement, then fine-tune on relevance labels. Both the teacher and student models could first pre-train on the ocean of cheap engagement labels (clicks, repins) to learn general query-Pin structure, then fine-tune on the higher-quality relevance labels. This combines behavioral scale with relevance precision, which will further reduce the dependency on Human Labels.

    Big picture. LLM-as-judge and LLM-as-teacher are fast becoming standard tools across the search, ads, and recommendation stack. For now, the winning pattern isn’t “put an LLM in the serving path” — it’s “use the LLM offline to manufacture training supervision at scale, then distill it into a cheap online model.” And as inference keeps getting cheaper, expect more classic ML problems to be improved by LLMs. Excited to see where LLM-driven search relevance goes next!
